# プロジェクト概要
- 大学の卒業チームプロジェクトとして開発した、ユーザーの食事、運動、目標達成をサポートする総合健康管理サービスです。
## 主なサービス機能 (Key Features)
- AI食事認識
  - TensorFlowによる食品画像認識機能と、詳細な栄養情報の提供。
- AIプランニング
  - OpenAI APIを活用した、ユーザーに合わせた最適な食事・運動計画の推薦。
- 記録・管理
  - モーション認識による運動姿勢の矯正と記録機能、AI推薦計画のチェックリスト提供。
- 統計表示	
  - プロフィール画面で運動・食事記録をチャート形式（Rechart）で表示。

# 技術的な課題と解決 (Challenge & Solution)
- 統計データ加工ロジック
- 解決策と学び
## 詳細と解決策
- 【問題】 Rechart表示に必要なデータをDBから取得後、同一日付の記録を全て集計・統合する処理を単一関数で実装したため、コードが長大化し、可読性と再利用性が低下。
- 【解決】 処理内容を細分化し、複数の機能別関数に分離（リファクタリング）。コードの繰り返しを削減し、可読性と保守性を大幅に向上させた。
- 【学び】 単なる機能実装だけでなく、コードの再利用性・保守性を考慮した設計の重要性を深く学んだ。

# チーム開発における貢献 (Collaboration & Process)
- 報連相の徹底: チーム開発において、情報伝達の重要性を痛感。
- コミュニケーション: Slackを主要な報告・連絡手段として活用し、各メンバーの進捗状況をリアルタイムで共有。
- 成果: 情報の透明性を確保することで、開発初期段階の認識齟齬を減らし、開発速度の向上に貢献。
